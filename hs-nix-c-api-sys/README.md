# hs-nix-c-api-sys

Raw Haskell FFI bindings to the [Nix C API](https://nix.dev/manual/nix/latest/c-api), auto-generated by [hs-bindgen](https://github.com/well-typed/hs-bindgen).

Covers the complete Nix C API surface across six libraries:

| C library | Haskell module | Purpose |
|---|---|---|
| `nix-util-c` | `Generated.Nix.Util` | Error handling, settings, string callbacks |
| `nix-store-c` | `Generated.Nix.Store` | Store operations, paths, derivations |
| `nix-expr-c` | `Generated.Nix.Expr`, `Generated.Nix.Value` | Expression evaluation, values, GC |
| `nix-fetchers-c` | `Generated.Nix.Fetchers` | Fetcher settings |
| `nix-flake-c` | `Generated.Nix.Flake` | Flake evaluation and locking |
| `nix-main-c` | `Generated.Nix.Main` | Plugin loading, logging |

Each C header generates four Haskell modules:

- `Generated.Nix.<Lib>` -- type definitions and constants
- `Generated.Nix.<Lib>.Safe` -- thread-safe FFI bindings
- `Generated.Nix.<Lib>.Unsafe` -- faster, non-thread-safe FFI bindings
- `Generated.Nix.<Lib>.FunPtr` -- function pointer wrappers for C callbacks

## Usage

```haskell
import Generated.Nix.Util (Nix_err (..), Nix_c_context)
import qualified Generated.Nix.Util.Safe as Util
import qualified Generated.Nix.Store.Safe as Store

main :: IO ()
main = do
  ctx <- Util.nix_c_context_create
  _ <- Util.nix_libutil_init ctx
  _ <- Store.nix_libstore_init ctx
  store <- Store.nix_store_open ctx "daemon" nullPtr
  -- ...
  Store.nix_store_free store
  Util.nix_c_context_free ctx
```

For a safe, ergonomic API with automatic resource management and error handling, use [hs-nix-c-api](../hs-nix-c-api/) instead.

## Generating bindings

The bindings are generated from the Nix C API headers using `hs-bindgen-cli`.
The Nix C libraries come from the [NixOS/nix](https://github.com/NixOS/nix) flake input.

Enter the dev shell and run:

```bash
nix develop
cd hs-nix-c-api-sys
rm -rf src/Generated binding-specs
mkdir -p binding-specs
./generate-bindings
```

The script processes headers in dependency order, using `--external-binding-spec` to prevent type duplication across modules.
Generated files are committed to the repository so the package builds without `hs-bindgen-cli` installed.

### Requirements for regeneration

- `hs-bindgen-cli` (provided by the flake's dev shell)
- `pkg-config` (provided by the flake's dev shell)
- Nix C API development headers (`nix-util-c`, `nix-store-c`, etc., provided by the `nix` flake input)
